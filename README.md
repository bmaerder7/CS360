Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?

The goal of this app was to create a place where the user can keep track of their weekly events and appointmnets that they want to keep track of. The app was designed to address user functions such as creating, modifying, or deleting custom events, as well as get SMS reminders in advance. 

What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

FOr this app, I wanted to promote simplicity and intuitive design for users ease of access. To accomplish this, I made the main screen the weekly view for the user, with an FAB of a plus in the bottom right to add an event, which most users are accustomed to in other applications. I also wanted modifying or deleting event as simple as clicking on the event time block, which brought up a modify or delete menu for the selected block, making this simple and easy for the user when prompted. 

How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?

For the coding of this app, I first started with the UI, creating all of the buttons textViews, and editTexts, the grid view, and all other UI components, with no working fucntions or transitions between screens. Then when this was complete, I already had a skeleton to incorperate functions and intents for each button and action, which was much simpler with an outline of the UI already built. I will definitely continue to approach coding in Andriod Studio in this way, as it helped break down a large code base into easy to work with, smaller portions. 

How did you test to ensure your code was functional? Why is this process important, and what did it reveal?

As I went through developing this app, nearly every funtion implimented or change to a UI element I made, I would stop and re-run the app on the emulator. This was extremely helpful in seeing what effect my last block of code or function had on the my application, sometimes leading to the entire app crashing unexpectedly, which although frustrating, was very helpful in learning what not to do with certain libraries and functions. 

Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?

This project was a bit challenging in that the instructions were a bit vague, but this also made it fun and interesting. The project called for user to be able to track their events, with the ability to add, modify or delete one, and their data must be stored in a database. Not much else was provided in terms of requirements or guidance with this project, and so as a developer it was a challenge but also an opportunity to get creative and envision what you believe would make a successful calendar app.

In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?

I feel that this project allowed me to show off my skills and knowledge in user-centered design. While I am still proud of my knowledge of dataflow and use of abstraction in this project too, I feel that my ability to see from the user's perspective, and include helpful features that make use of the app easier to navigate for new users, yet still including enough customization to keep them using the app time and time again.
